# Frontend Design

## Core Workflow

1.  **Define Layout:** Choose between standard tiered pricing or grid-based dashboard layouts.
2.  **Select Components:** Identify the necessary UI elements (e.g., toggles, stats cards, SVG charts).
3.  **Apply Visual Polish:** Use consistent spacing, typography (JetBrains Mono/Inter), and high-contrast minimal aesthetics.
4.  **Implementation:** Provide React/Tailwind or Vanilla CSS code based on the user's preference.

## Design Patterns

- **Pricing Pages:** Refer to [references/pricing-patterns.md](references/pricing-patterns.md) for layouts and components.
- **Dashboards:** Refer to [references/dashboard-patterns.md](references/dashboard-patterns.md) for monitoring and stats layouts.

## Guidelines

- **Typography:** Use `JetBrains Mono` for technical dashboards and `Inter` or system-sans for general UI.
- **Spacing:** Maintain a 4px/8px grid for consistent padding and margins.
- **Color:** Stick to a high-contrast palette (Black/White/Gray) with a single accent color for primary actions.
