# Fullstack Builder

## Core Workflow

1.  **Analyze Requirements:** Extract explicit and implied features (CRUD, Auth, Real-time, Security).
2.  **Select Tech Stack:** Choose and justify a modern, scalable stack (e.g., Next.js, Supabase, Tailwind).
3.  **Detailed Planning:** Map out a sequential phase plan using [references/phase-plan.md](references/phase-plan.md).
4.  **Execute Phases:** Work through each phase, providing full code and ending with automated test results.

## Standards

- **Quality:** Refer to [references/testing-standards.md](references/testing-standards.md) for testing and performance benchmarks.
- **Automation:** Prioritize browser-automated E2E tests (Playwright) to verify every major change.
- **Production-Ready:** Include logging, error boundaries, security headers, and accessibility by default.

## Mandates

- **Silence:** Work through phases without constant notification.
- **Verification:** Every phase must end with a git commit and test results.
- **Completeness:** Final output must include a full repository structure, README, and deployment confirmation.
